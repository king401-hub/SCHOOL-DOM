# QR Code-Based Teacher Attendance System

## Overview

This comprehensive attendance system allows school administrators to manage teacher attendance using QR codes. Teachers scan a single, static QR code generated by the admin, and the system automatically records their presence with timestamp and device information.

## Features

### 📱 Admin Features
- **QR Code Generation**: Generate a single, static QR code for the school
- **QR Code Management**: View, regenerate, and download QR codes as PNG
- **Real-time Dashboard**: See all teachers who marked attendance today
- **Date-based Filtering**: View attendance records for any past date
- **Attendance Summary**: Monthly overview of all teachers' attendance
- **Teacher History**: Detailed attendance history for individual teachers
- **Export Ready**: API endpoints for data integration and reporting

### ✓ Teacher Features
- **QR Scan**: Simple QR code scanning to mark attendance
- **Authentication**: Secure login required before marking attendance
- **Once Per Day**: Automatic prevention of duplicate daily attendance
- **Instant Confirmation**: Real-time feedback on attendance marking
- **History Tracking**: View personal attendance records

### 🔒 Security Features
- **Token-based Authentication**: JWT authentication for all endpoints
- **Role-based Access Control**: Strict enforcement of admin/teacher roles
- **Tenant Isolation**: Multi-tenant support with data isolation
- **Device Tracking**: Records IP address and device information
- **Duplicate Prevention**: Ensures only one attendance per teacher per day
- **Secure QR Token**: Cryptographically secure token generation

## API Endpoints

### QR Code Management (Admin Only)

#### Generate/Get QR Code
```
POST /api/attendance/qr-code/generate/
GET /api/attendance/qr-code/get/
```

**Request (POST):**
```json
{
  "regenerate": false,
  "notes": "Generated for Q1 2024"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "token": "secure_token_string",
    "is_active": true,
    "created_at": "2024-01-15T10:00:00Z",
    "today_attendance_count": 42
  }
}
```

#### Download QR Code
```
GET /api/attendance/qr-code/download/
```

**Response:** PNG image file (application/png)

### Teacher Attendance Marking

#### Scan QR Code (Public)
```
GET /api/attendance/scan/{token}/
POST /api/attendance/scan/{token}/
```

**GET Response:**
```json
{
  "success": true,
  "message": "QR code verified. Please log in as teacher.",
  "token": "secure_token"
}
```

**POST Request (Authenticated):**
```
Authorization: Bearer {jwt_token}
Content-Type: application/json
```

**POST Response:**
```json
{
  "success": true,
  "message": "Attendance marked successfully at 09:45:30",
  "data": {
    "id": "uuid",
    "teacher": "uuid",
    "teacher_details": {
      "id": "uuid",
      "email": "teacher@school.com",
      "full_name": "John Doe"
    },
    "attendance_date": "2024-01-15",
    "check_in_time": "2024-01-15T09:45:30Z",
    "check_in_time_formatted": "09:45:30",
    "status": "present"
  }
}
```

#### Check Attendance Status (Teacher)

**Response:**

## Database Models

### AttendanceQRCode
- Single QR code per school tenant
- Contains secure token
- Tracks creation and updates
- Can be activated/deactivated

### TeacherAttendance
- Stores individual attendance records
- One record per teacher per day (unique constraint)
- Records check-in time, IP address, device info
- Supports status tracking (present, late, absent)

### AttendanceReport
- Monthly or custom period reports
- Attendance statistics (present, absent, late days)
- Attendance percentage calculation
- Generated and tracked by admin

## Usage Workflow

### For Administrators

1. **Generate QR Code:**
   - Go to Admin Dashboard → QR Code Management
   - Click "Generate QR Code"
   - Click "Preview" to see the QR code
   - Click "Download PNG" to save for printing

2. **Display QR Code:**
   - Print and display at office/entrance
   - Share digital copy with teachers
   - Can be placed on digital displays

3. **Monitor Attendance:**
   - Go to Attendance Dashboard
   - Select date to view records
   - Click "Refresh" for real-time updates
   - Export data as needed

4. **Generate Reports:**
   - View Monthly Summary Report
   - Analyze teacher attendance patterns
   - Export for further analysis

### For Teachers

1. **Scan QR Code:**
   - Open camera/QR scanner app
   - Scan the displayed QR code
   - App opens attendance page

2. **Authenticate:**
   - Log in with teacher credentials if not already logged in
   - System verifies teacher role

3. **Mark Attendance:**
   - Click "Mark Attendance" button
   - System records check-in time automatically
   - Receive confirmation

4. **View History:**
   - Teachers can see their own attendance history
   - Check last check-in time and date

## Security Considerations

### Token Security
- Tokens are 64-character cryptographically secure strings
- Tokens are unique per school tenant
- Can be regenerated anytime
- Invalid tokens return 401 Unauthorized

### Authentication
- All marking endpoints require JWT authentication
- Teacher role is strictly verified
- Tenant isolation prevents cross-tenant access

### Data Protection
- IP address recorded for audit trail
- Device information captured for security
- Timestamps use UTC timezone
- All data encrypted at rest (configurable)

### Rate Limiting (Optional Configuration)
Add to Django settings:
```python
REST_FRAMEWORK = {
    'DEFAULT_THROTTLE_CLASSES': [
        'rest_framework.throttling.AnonRateThrottle',
        'rest_framework.throttling.UserRateThrottle'
    ],
    'DEFAULT_THROTTLE_RATES': {
        'anon': '100/hour',
        'user': '1000/hour'
    }
}
```

## Error Handling

### Common Errors and Solutions

| Error | Cause | Solution |
|-------|-------|----------|
| Invalid authorization key | QR code not found | Regenerate QR code |
| Only teachers can mark attendance | Non-teacher account | Log in as teacher |
| Already marked attendance today | Duplicate attempt | Check in is recorded |
| Not authenticated | No JWT token | Login required |
| Unauthorized access | Different tenant | Verify school/tenant |

## Reporting & Analytics

### API for Custom Reports

```python
from attendance.models import TeacherAttendance
from django.utils import timezone
from datetime import timedelta

# Get attendance for past 30 days
start_date = timezone.localdate() - timedelta(days=30)
records = TeacherAttendance.objects.filter(
    attendance_date__gte=start_date
).select_related('teacher')

# Calculate statistics
present_count = records.filter(status='present').count()
late_count = records.filter(status='late').count()
```

### Export to CSV

```python
import csv
from attendance.models import TeacherAttendance

def export_attendance_csv(tenant, start_date, end_date):
    records = TeacherAttendance.objects.filter(
        tenant=tenant,
        attendance_date__range=[start_date, end_date]
    )
    
    response = HttpResponse(content_type='text/csv')
    response['Content-Disposition'] = 'attachment; filename="attendance.csv"'
    
    writer = csv.writer(response)
    writer.writerow(['Teacher', 'Email', 'Date', 'Check-in Time', 'Status'])
    
    for record in records:
        writer.writerow([
            record.teacher.get_full_name(),
            record.teacher.email,
            record.attendance_date,
            record.check_in_time.strftime('%H:%M:%S'),
            record.status
        ])
    
    return response
```

## Troubleshooting

### QR Code Not Generating
1. Check `qrcode` package is installed: `pip install qrcode[pil]`
2. Verify FLUTTERWAVE_SECRET_KEY is configured
3. Check logs for specific errors

### Attendance Not Recording
1. Verify JWT token is valid
2. Check user role is 'teacher'
3. Verify user belongs to correct tenant
4. Ensure it's before 11:59 PM (server time)

### Dashboard Not Loading
1. Clear browser cache
2. Verify admin role
3. Check network requests in DevTools
4. Verify Django server is running
